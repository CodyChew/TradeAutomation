Manual Refresh Workflow

Default command: .\venv\Scripts\python.exe scripts\build_lpfs_live_weekly_performance.py --latest

The script reads over SSH, fingerprints the inputs, and prints already up to date without rewriting files when the latest report already matches current journals, state, benchmarks, and git heads. Use --force only when intentionally regenerating the same evidence.

Metric Glossary

Realized DD
Closed-trade equity drawdown only. It ignores open trade risk until a position exits.
Risk-Reserved DD
Live-account stress drawdown that subtracts full open risk while positions are active.
Max Reserved Risk
Largest total open risk reserved at one time across accepted portfolio trades.
Worst Month
Largest calendar-month loss in account-percent terms for the tested schedule.
Return/DD
Total return divided by risk-reserved drawdown where available; higher is more efficient.
Avg R / PF
Average net R per trade and profit factor. Use both with trade count before trusting a row.
Read-only monitor. No live configs, VPS runtime state, journals, orders, positions, or scheduled tasks are changed.
