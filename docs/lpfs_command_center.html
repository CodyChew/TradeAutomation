Ownership And Refresh Workflow

The strategy agent owns noticing weak performance, checking whether evidence is complete, converting repeated weak cohorts into offline research candidates, rejecting unsupported hypotheses, and asking for missing data or tooling before recommending live changes. This dashboard should make that ownership visible instead of hiding it inside dense weekly tables.

Role coverage

  • Main Orchestrator: routes material dashboard changes through change gate.
  • LPFS Strategy Improvement Agent: owns candidate questions, interpretation, and next action.
  • Documentation and Workflow Agent: owns source-of-truth clarity and generated-page hygiene.
  • Repo Auditor: checks stale claims and generated artifact scope.
  • Reliability Reviewer: reviews live-status wording and operational boundaries.
  • Independent Issue Verifier: verifies issue/data truth and production-impact claims.
  • QA/Test Engineer: owns command-center tests and link/contract coverage.
  • Risk Manager: checks risk framing and no-live-change boundaries.
  • Strategy Trader: checks trader-facing interpretation and candidate wording.
  • Data Engineer: checks evidence lineage, hashes, and data-quality fields.
  • Dashboard/UI owner: proposed lightweight role for first-screen hierarchy, caveats, readability, and refresh workflow.
