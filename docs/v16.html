V16 Bid/Ask Execution Realism

Execution realism

Decision Brief

V16 result: bid/ask realism is not a material regression; do not change live stops yet.

  • No-buffer V16: 12,917 trades versus 13,012 baseline trades, total 1,535.2R versus 1,512.3R, PF 1.270 versus 1.265.
  • No-buffer V16 missed 95 baseline trades, changed 284 exit reasons, and still passed V15 practical bucket filters.
  • The 1.5x spread-buffer variant had the best raw total R at 1,587.1R and a strong high-return bucket row, but it changed 722 exit reasons and 493 win/loss signs.
  • Because buffer behavior is more invasive than bid/ask realism itself, keep live FS structure stops unchanged until a focused buffer-specific validation is reviewed.

Next: Keep no-buffer live stops for now. If improving stop robustness becomes a priority, run a focused buffer validation by timeframe and symbol group.

What This Tests

Does broker-realistic bid/ask triggering materially weaken the V15 baseline, and should Force Strike stops include a spread-based buffer?

Setup Tested

Research-only rerun of the current LP3, H4/H8/H12/D1/W1, 0.5 pullback, FS structure stop, 1R target, fixed 6-bar pullback model. OHLC is treated as Bid and Ask is approximated from each candle's stored spread_points and point.

How To Read This Page

Start with Decision Read, then compare raw R deltas and V15 bucket sensitivity per buffer. Do not change live stops until the no-buffer and buffer rows are reviewed together.

Use / Do Not Use

Keep current live stop placement unchanged for now. Use V16 as evidence that bid/ask mechanics are acceptable, and treat spread buffers as a promising follow-up rather than an immediate live-rule change.

Conclusion: Bid/ask realism did not weaken the current baseline. The no-buffer model slightly improved raw total R while missing less than 1% of baseline trades. A 1.5x spread buffer improved some bucket rows, but it changes outcomes materially enough that live stops should stay no-buffer until a focused buffer follow-up confirms it.

Execution Rules Tested

  • OHLC is interpreted as Bid; Ask OHLC is Bid OHLC plus that bar's stored spread_points * point.
  • Long entry requires Ask low <= BUY LIMIT entry. Long stop/target uses Bid touches.
  • Short entry requires Bid high >= SELL LIMIT entry. Short stop/target uses Ask touches.
  • Same-bar TP/SL conflict remains stop-first.
  • Stop buffers test 0.0x, 0.5x, 1.0x, 1.5x, and 2.0x signal-candle spread, with target recalculated to 1R.

Metric Glossary

Realized DD
Closed-trade equity drawdown only. It ignores open trade risk until a position exits.
Risk-Reserved DD
Live-account stress drawdown that subtracts full open risk while positions are active.
Max Reserved Risk
Largest total open risk reserved at one time across accepted portfolio trades.
Worst Month
Largest calendar-month loss in account-percent terms for the tested schedule.
Return/DD
Total return divided by risk-reserved drawdown where available; higher is more efficient.
Avg R / PF
Average net R per trade and profit factor. Use both with trade count before trusting a row.
